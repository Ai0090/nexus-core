◈

Quantum-Immune Security (PQC Ready)

While legacy chains rely on vulnerable elliptic curves, TET utilizes a hybrid signature model (Ed25519 + NIST ML-DSA-44). It is engineered for resilience against future quantum decryption (Shor-class attacks on ECC).

◇

Zero-Honeypot Architecture (Pure Non-Custodial)

We hold no keys. Wallets are strictly client-side, encrypted locally using Argon2id + AES-256-GCM. A server breach yields zero funds.

▣

Atomic Ledger & Replay Immunity

Built on Rust, every state change is strictly atomic. Nonce-based hybrid signatures make double-spend and replay attacks structurally impossible at the verification layer.

✦

Hyper-Deflationary AI Compute

Tradable Energy Token aligns economic incentives so compute becomes a distributed network: independent operators contribute capacity, users pay in TET for real work, and surplus value flows back to contributors — weakening extractive concentration and widening who can participate in the AI economy.

How it works

The utility loop closes the circuit between demand and reward. Users spend TET for services — for example, sending a prompt through the utility path. The network routes work to participating workers; heartbeat-registered Genesis workers earn from the designated worker pool according to protocol rules. Spend, route, verify, earn — a closed loop designed for transparency and sustained participation.

Quantum security

Settlement and wallet authentication use a verified hybrid model: classical Ed25519 plus NIST ML-DSA-44 on shared canonical payloads, so identities remain credible even if large-scale quantum adversaries threaten elliptic-curve assumptions alone.

Running a Heavy Worker Node (CLI)

For enterprise compute providers and developers, TET allows direct P2P network participation via the Rust CLI.

Step 1: Clone the repository.
git clone <REPO_URL>\ncd \"Nexus Network/tet-core\"
Step 2: Configure your TET_API_KEY (if required for legacy bypass) or ensure your wallet is active.
export TET_API_KEY=\"YOUR_KEY\"
Step 3: Run the production engine.
cargo run --release --bin TET-Core

CLI workers process heavy ML/AI tasks and earn maximum TET routing/compute fees.
